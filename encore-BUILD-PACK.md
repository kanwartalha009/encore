## 1. App

- Name: **Encore** — Preorder & back-in-stock for EU fashion — per-market, never oversells
- Description: Turn out-of-stock and not-yet-released products into captured demand. Merchants enable preorder or a back-in-stock waitlist on any product in under two minutes — works inside the store's existing Shopify discounts, never oversells, flat transparent pricing, and preorder by market (in-stock here, preorder there). Shoppers reserve with pay-now, a deposit, or charge-later (incl. EU BNPL) via Shopify's Selling Plan API. The demand-capture front-end of the ARRS recovery engine: waitlist captures run through behavioural recovery, not a generic email blast.
- Slug: `encore` · Status: DRAFT · Pricing: FREEMIUM
- Shopify org: Nova Apps (#4218705) · App ID: 88241007
- Repo: `github.com/nova-apps/encore` · Modules: backend, storefront-widget, flow, customer-account
- Plans: Free free · Growth $19.00/mo · Scale $49.00/mo
- Problem: EU fashion and footwear brands run constant drops and restocks with limited inventory across multiple markets, and lose revenue three ways: sold-out products stop capturing demand, stock lands in one market before another (but stores can't sell 'in-stock here, preorder there'), and incumbent preorder apps are unreliable — they oversell, drop order tags, and break site-wide discounts.
- User: EU DTC fashion, footwear, jewelry and accessories brands on Shopify (Basic→Plus) selling across multiple markets and currencies. The operator/founder enables it in minutes; merchandising/ops consumes the order tags, ship dates, and demand signal.
- Outcome: Enable preorder or a back-in-stock waitlist on any product in under two minutes — never oversell, never lose a tag, correct inside existing discounts, preorder by market, and captured demand flows into ARRS recovery that converts better than a generic back-in-stock email.

## 2. Stack & conventions (non-negotiable)

- Scaffold: `shopify app init` → **React Router template** (Shopify's recommended path); Polaris web components + App Bridge for all App Home UI.
- Build pages from Polaris **page patterns**: templates (homepage/settings/index/detail) + compositions (setup-guide, data-table, empty-state…). Match Built for Shopify standards.
- Auth: **embedded app + token exchange** — App Bridge mints a ~60s session token (signed with the app secret); exchange it for an access token to call the Admin API. Managed install applies scopes from `shopify.app.toml`. No cookies.
- Backend hosted at `http://encore.nova-platform.localhost:3003` — OAuth `/api/auth`, webhooks `/api/webhooks` (verify X-Shopify-Hmac-Sha256 with the app secret).
- Database: per-app Postgres via Prisma — schema at `apps/app-admin/db/encore/schema.prisma`, env `APP_DB_URL__ENCORE`. Never touch other apps' DBs or the platform DB.
- Every shop-scoped table carries `shopId`; GDPR redact purges by it.
- OAuth scopes: `read_products`, `write_products`, `read_inventory`, `read_locations`, `read_orders`, `write_orders`, `read_markets`, `read_customers`, `read_themes`.

## 5. Nova platform integration (wire this exactly)

This app is one tenant of the Nova Apps Platform. Two signed channels keep the platform's installs, billing, and commissions correct — without them, revenue and agency commissions never appear. `NOVA_API` is the platform API base URL.

### 5.1 Install-confirm callback
After OAuth / token-exchange completes, call `POST {NOVA_API}/v1/internal/installations/confirm` — HMAC-signed with `NOVA_INSTALL_CONFIRM_SECRET`. Body: `{ shopDomain, appSlug: "encore", planName?, installedAt }`. This flips the Installation to ACTIVE and locks the immutable agency referral attribution.

### 5.2 Webhook forwarding to ingress
Handle these app-locally **and** forward them (HMAC-signed with `NOVA_INGRESS_HMAC_SECRET`) to `{NOVA_API}/v1/webhooks/shopify/encore`:
- `app_subscriptions/update`, `app/uninstalled` — billing + lifecycle (this is how charges → commissions are derived in the platform ledger).
- `customers/data_request`, `customers/redact`, `shop/redact` — GDPR; respond **200**, return **401** on an invalid HMAC, complete within 30 days.

### 5.3 Billing sync
- AppPlan rows (§4) are the plan constants; Billing API mutations use them.
- Honor `Installation.planOverride` (per-store comp/discount) at subscription creation.

### 5.4 Per-app database
Isolated Postgres via Prisma at `apps/app-admin/db/encore/schema.prisma` (env `APP_DB_URL__ENCORE`) — one database + one migration history for this app only.

### 5.5 External integrations
- **ARRS recovery engine** (outbound) — Encore owns demand capture; ARRS owns recovery. Waitlist/preorder/abandonment signals are emitted as events; ARRS applies behavioural segmentation + localized AI recovery (the differentiator vs a generic back-in-stock email).
  - Env: `ARRS_API_BASE`, `ARRS_API_KEY`
  - Events: `waitlist.signup`, `preorder.intent`, `preorder.cart_abandoned`, `restock.detected`

### 7.3 Webhook handlers

- `orders/create` → Detect preorder line properties; tag order + line 'preorder' (idempotent, verified post-write); write ship-date metafield; increment DemandSignal; emit preorder-intent to ARRS. (writes: order tags/metafields (Admin), DemandSignal)
- `orders/paid` → For deposit / charge-later, create or confirm the ChargeSchedule from the selling-plan mandate. (writes: ChargeSchedule)
- `inventory_levels/update` → Restock detection: fire ARRS recovery for waitlisted variants back in stock for a market; reconcile preorder eligibility; enforce no-oversell. (writes: WaitlistEntry, PreorderConfig)
- `products/update` → Re-scan OOS state; keep PreorderConfig + selling plans in sync with variant changes. (writes: PreorderConfig)
- `app/uninstalled` → Mark shop uninstalled, schedule data purge, forward to platform ingress. (writes: Shop)
- `app_subscriptions/update` → Forward to platform ingress (billing source of truth). (writes: —)
- `customers/data_request | customers/redact | shop/redact` → GDPR: export/purge waitlist PII by shopId/email; always 200 within 5s. (writes: all shop-scoped tables)

### 7.4 Scheduled jobs

- `charge-later-collect` (hourly) — Collect due balances via the vaulted mandate; idempotent + retry-safe; append to retryLog. Sends balance-due reminder w/ payment link where configured (P1).
- `waitlist-dispatch` (every 5 min) — Durable queue: deliver back-in-stock recovery via ARRS; confirm delivery and set notifiedAt — never best-effort (§13).
- `inventory-reconcile` (every 15 min) — Reconcile preorder eligibility vs real location inventory per market; flip Buy/Preorder states; guard the no-oversell limit.
- `demand-rollup` (daily 02:00 shop-local) — Aggregate preorder intent + waitlist into DemandSignal by variant/size/market.
- `purge-uninstalled` (daily) — Hard-delete shop data 48h after uninstall (GDPR).

## 8. Reliability bar (definition of done — "must never")

A version that violates any of these does not ship:

- [ ] Never oversell beyond the configured preorder limit
- [ ] A preorder/backorder order is never created untagged
- [ ] Preorder items never silently break a store's discounts
- [ ] Changing one setting never alters an unrelated setting
- [ ] A configured back-in-stock notification never silently fails to fire
- [ ] Never a broken checkout when a payment method is unsupported (graceful fallback)
- [ ] Never continue billing after uninstall
